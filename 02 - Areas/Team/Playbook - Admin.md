## 🧩 My Orchestration Patterns

### Pattern A — Meeting Coordination
**ใช้เมื่อ:** Any important meeting (leadership, board, customer)
**Chain:**
```
1. Schedule (calendar + room / zoom)
2. Send pre-read 24h ahead
3. Attend / take notes
4. `meeting-analyzer` → TL;DR, decisions, action items, owners + due
5. Send recap within 2h
6. Track action items → nudge owners before due
7. `decision-logger` → if strategic
```

### Pattern B — New Hire Onboarding
**ใช้เมื่อ:** New employee starts
**Chain:**
```
1. Prep: laptop, accounts, access, desk
2. Day-1 orientation (CEO welcome, tools, policies)
3. Assign buddy + 1:1 with manager
4. Week-1 check-in (support, blockers)
5. Week-2 — review Role Playbook Template kickoff
6. Day-30 — pulse survey
7. `change-management` → log feedback → improve process
```

### Pattern C — Vendor / Contract Flow
**ใช้เมื่อ:** New vendor or contract renewal
**Chain:**
```
1. Collect requirements from requester
2. 3 vendor quotes (if > threshold)
3. `contract-and-proposal-writer` → draft terms review
4. บัญชี → finance review + approval
5. CEO → sign if > threshold
6. File + set renewal reminder
7. `decision-logger` → vendor decision
```

## 🤝 Handoff Points

**รับงานจาก:**
- **CEO** → priorities, scheduling requests, strategic support
- **All team** → meeting requests, doc / vendor needs
- **บัญชี** → invoices for routing, expense approvals
- **New hires** → onboarding questions

**ส่งงานให้:**
- **ทุกคน** → meeting recaps, calendar invites
- **บัญชี** → processed invoices, expenses, receipts
- **CEO** → daily brief, weekly ops report
- **External vendors** → coordinated requests, approved contracts

**Review กับ:**
- **CEO** — daily standup (short), weekly ops review
- **บัญชี** — daily invoice handoff, weekly spend review
- **Leadership** — weekly staff meeting coordination
