## 🧩 My Orchestration Patterns

### Pattern A — New Page SEO Workflow
**ใช้เมื่อ:** Marketing / BA ต้องการ publish new page/post
**Chain:**
```
1. `competitive-teardown` → SERP analysis + gaps
2. Keyword research (primary + secondary + LSI)
3. Brief to `content-creator` (outline + word count + intent)
4. Review draft → `copywriting` (meta + H1-H3)
5. `schema-markup` → structured data (Article / Product / FAQ)
6. `seo-audit` → on-page check (pre-publish)
7. Publish → submit to GSC
8. Monitor rankings → iterate
```

### Pattern B — Technical Audit
**ใช้เมื่อ:** Monthly or after major release
**Chain:**
```
1. `seo-audit` → crawl (indexation, canonicals, 404s)
2. `performance-profiler` → CWV check
3. `schema-markup` → validate structured data
4. `a11y-audit` → cross-check (affects SEO)
5. Report prioritized fixes → Sr FE
6. Track fix SLA
```

### Pattern C — Content Refresh
**ใช้เมื่อ:** Existing content losing rankings
**Chain:**
```
1. `campaign-analytics` → identify declining pages
2. `competitive-teardown` → who overtook + why
3. `content-creator` → refresh outline (update, expand, deepen)
4. Update + re-submit to GSC
5. Monitor re-rank trajectory
```
